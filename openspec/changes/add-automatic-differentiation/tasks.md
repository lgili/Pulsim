## Phase 1: AD scalar bridge
- [x] 1.1 [`core/include/pulsim/v1/ad/ad_scalar.hpp`](../../../core/include/pulsim/v1/ad/ad_scalar.hpp) exposes `ADReal = Eigen::AutoDiffScalar<Eigen::VectorXd>` (forward-mode, dynamic derivative-vector dimension matching the device's terminal count). Header-only, lives in the `pulsim::v1::ad` namespace, depends on `unsupported/Eigen/AutoDiff` already shipped with our Eigen 3.4 install.
- [x] 1.2 `ad::seed_from_values(span<const Real>)` constructs a `std::vector<ADReal>` where each output `i` carries `value = values[i]` and `derivatives = e_i`. An overload taking `initializer_list<Real>` simplifies test/device-residual call sites.
- [x] 1.3 `ad::jacobian_row(residual)` extracts `∂residual / ∂x_i` for a single residual scalar; `ad::stack_jacobian(span<const ADReal>)` stacks per-row gradients into an `Eigen::MatrixXd` for the multi-residual case (defensive against AD scalars with empty derivative vectors — produced when arithmetic only involves constants).
- [x] 1.4 Unit test in [`test_ad_scalar.cpp`](../../../core/tests/test_ad_scalar.cpp): scalar `f(x) = sin(x) · exp(x)` evaluated via AD matches analytical `f'(x) = exp(x) · (cos(x) + sin(x))` exactly.
- [x] 1.5 Unit tests in the same file: linear residual `i = G · (v_a − v_b)` produces Jacobian row `[+G, −G]` exactly; Shichman-Hodges drain current `id = 0.5·kp·(vov)²·(1+λ·vds)` matches centered finite-difference within 1e-4 absolute and matches the analytical partial derivatives exactly; `stack_jacobian` builds a 2×3 matrix with the correct entries for a mixed linear+bilinear residual pair.

### Plumbing notes for downstream phases
- AD chain breaks if you construct an `ADReal` from a Real scalar (e.g. `ADReal{0.5}`): the resulting derivatives vector is empty and subsequent multiplication zeros the gradient. Workaround: keep Real literals as Real; `Real * ADReal` propagates correctly through Eigen's overloaded `operator*`.
- For mixed-mode device residuals authored as `template <typename Scalar> S residual_at(...)`, the device should accept its terminal voltages as `Scalar` and its physical parameters as `Real` — not as `Scalar` — so constants don't accidentally erase derivatives in the Real instantiation path.
- `unsupported/Eigen/AutoDiff` provides ADL-found overloads of `sin`, `cos`, `exp`, `sqrt`, etc. Tests should `using std::sin;` etc. (or call them unqualified) so the AD overload is found alongside the Real one.

## Phase 2: Device contract migration (nonlinear)
- [x] 2.1 `IdealDiode` now exposes `template<typename S> S forward_current_behavioral(S v_anode, S v_cathode) const` — single source of truth for the Behavioral residual, consumed by both the manual stamp and the AD path. Templated on the scalar type so the same expression compiles for `Real` and `ADReal`. Coefficients stay as `Real` per the Phase 1 plumbing note (constructing `S{constant}` would erase the derivative chain).
  - MOSFET / IGBT / VoltageControlledSwitch — deferred to follow-up sessions; same pattern (extract residual into a templated helper, share with manual stamp).
- [x] 2.2 (MOSFET) `drain_current_behavioral<S>` lands in [`mosfet.hpp`](../../../core/include/pulsim/v1/components/mosfet.hpp) — Shichman-Hodges with all three regions (cutoff / triode / saturation) and NMOS/PMOS sign handling. `stamp_jacobian_via_ad` replicates the manual stamp's Norton companion form (`i_eq = id − ∇id · x`) — but in the actual coordinate basis rather than the manual's sign-flipped `vgs_local/vds_local`. Cross-validation in [`test_ad_mosfet_stamp.cpp`](../../../core/tests/test_ad_mosfet_stamp.cpp) covers 6 NMOS op-points (manual ≡ AD within 1e-12) and 3 PMOS op-points (AD ≡ centered FD within 1e-6). **Side discovery**: the legacy manual stamp has a pre-existing sign bug in the PMOS `i_eq` linearization terms — it computes `id − gm·vgs_local − gds·vds_local` using sign-flipped local coordinates without reflecting the chain-rule sign back into the linearization terms. AD computes the correct offset; PMOS validation falls back to FD until the legacy stamp is corrected (separate concern).
- [x] 2.2 (IGBT) `collector_current_behavioral<S>` lands in [`igbt.hpp`](../../../core/include/pulsim/v1/components/igbt.hpp) — region selection via `(vge > vth) ∧ (vce > 0)` with `g ∈ {g_on, g_off}` per region. `stamp_jacobian_via_ad` uses the same Norton-Taylor form as MOSFET. Cross-validation in [`test_ad_igbt_stamp.cpp`](../../../core/tests/test_ad_igbt_stamp.cpp) covers 5 op-points (cutoff, on-below-sat, on-above-sat, Vce ≤ 0, asymmetric emitter reference) — manual ≡ AD within 1e-12 absolute on every J entry and `f` row. The legacy stamp's Vce_sat saturation branch (which collapses algebraically to the same `g · vce` in the on-state) carries through identically.
- [x] 2.2 (VoltageControlledSwitch) `switch_current_behavioral<S>` lands in [`voltage_controlled_switch.hpp`](../../../core/include/pulsim/v1/components/voltage_controlled_switch.hpp) — tanh-smoothed conductance `g(v_ctrl) = g_off + (g_on − g_off) · sigmoid((v_ctrl − v_th)/hysteresis)` followed by `i = g · (v_t1 − v_t2)`. `stamp_jacobian_via_ad` uses **standard form** (matching the manual VCSwitch stamp's `f[t1] += i_sw` convention, which differs from MOSFET/IGBT's Norton form). Cross-validation in [`test_ad_vcswitch_stamp.cpp`](../../../core/tests/test_ad_vcswitch_stamp.cpp) covers 8 op-points (saturated on/off, exactly at threshold where dg/dv_ctrl peaks, mid-transition above/below, asymmetric reference, zero v_sw, custom hysteresis) — manual ≡ AD within 1e-12 absolute. **Phase 2.2 fully complete: all four nonlinear devices (Diode, MOSFET, IGBT, VCSwitch) now have AD-derived stamps validated against the manual baseline.**
- [x] 2.3 `IdealDiode::stamp_jacobian_via_ad(J, f, x, nodes)` lands in [`ideal_diode.hpp`](../../../core/include/pulsim/v1/components/ideal_diode.hpp). Seeds `(v_anode, v_cathode)` with `ad::seed_from_values`, evaluates `forward_current_behavioral<ADReal>`, extracts `i_diode = i_ad.value()` and the partials `(∂i/∂v_anode, ∂i/∂v_cathode) = i_ad.derivatives()`, then stamps the symmetric 2×2 Jacobian and the residual contribution. The same shape applies to higher-terminal devices (3 derivatives for MOSFET/IGBT, etc.).
- [x] 2.4 Build-time toggle landed: new CMake option `PULSIM_USE_AD_STAMP` (default OFF) propagates the compile definition `PULSIM_USE_AD_STAMP=1`. Each of the four devices' `stamp_jacobian_impl` Behavioral branch wraps both paths in `#ifdef PULSIM_USE_AD_STAMP` — manual stays the default (preserves byte-for-byte legacy behavior); opt-in flips Diode/MOSFET/IGBT/VCSwitch to the AD-derived stamp at once. Both modes verified against the full test suite:
  - Default (`PULSIM_USE_AD_STAMP=OFF`): 297 cases / 4525 assertions ✓
  - Opt-in   (`PULSIM_USE_AD_STAMP=ON`): 297 cases / 4525 assertions ✓
  Identical assertion counts mean the AD path produces results indistinguishable from manual at the test-suite resolution. `stamp_jacobian_via_ad` was upgraded from `const` to non-const so it mirrors the manual stamp's `pwl_state_` mutation — downstream `is_conducting()` callers see consistent runtime state regardless of stamp path. The eventual unconditional flip to AD-as-default depends on the legacy PMOS sign-bug fix landing first (logged as a separate task).
- [x] 2.5 Cross-validation test for IdealDiode landed in [`test_ad_diode_stamp.cpp`](../../../core/tests/test_ad_diode_stamp.cpp): 8 operating points covering deep-reverse, mid-reverse, threshold, mid-forward, strong-forward, asymmetric reference, and both sharp branches (`v_smooth = 0` reverse / forward). Manual and AD stamps agree on every J entry and `f` row to within 1e-12 absolute. **Gate G.1 satisfied for diode**; remaining devices follow once 2.2 lands.

## Phase 4: Validation layer
- [x] 4.1 `pulsim::v1::ad::validate_nonlinear_jacobians(circuit, operating_points, abs_tol)` lands in [`ad/validate.hpp`](../../../core/include/pulsim/v1/ad/validate.hpp). Walks every nonlinear device in the circuit via `std::visit`, dispatches per device type, calls `device.stamp_jacobian` (which uses the build-selected manual-or-AD path), and compares the canonical "current-out" J row against centered finite differences on the device's templated residual helper (`forward_current_behavioral` / `drain_current_behavioral` / `collector_current_behavioral` / `switch_current_behavioral`). Returns a `std::vector<JacobianMismatch>` carrying `{device_name, device_type, op_point_index, local_row, local_col, stamp_value, fd_value, abs_delta}` for every entry above tolerance — empty result means the build agrees with FD everywhere.
  - Free function rather than a `Simulator::validate_jacobians` member: per-device validation needs the templated residual helpers, which are not exposed through the runtime polymorphic Simulator interface. Tests / CI call the free function directly with a small set of curated operating points.
- [ ] 4.2 `--validate-jacobians` CLI flag / `SimulationOptions.validate_jacobians_at_init` — deferred. The free function in 4.1 is the underlying engine; the CLI/options surface is a thin wrapper that lives best alongside the catalog tier work (`add-catalog-device-models`) once `SimulationOptions` is broken into smaller config blocks.
- [x] 4.3 `JacobianMismatch` already carries the structured diagnostic fields the proposal calls for (device name, device type, op-point index, local-J coordinates, stamp value, FD value, absolute delta). Test assertions log them via Catch2 INFO so the CI message identifies exactly which device drifted.
- [x] 4.4 CI test [`test_ad_validate.cpp`](../../../core/tests/test_ad_validate.cpp): builds a circuit with all four nonlinear device types and three curated operating points spanning their distinct regions, calls `validate_nonlinear_jacobians` with `abs_tol = 1e-5`, asserts the returned vector is empty. A second case pins the contract that an unrealistically tight tolerance (1e-15, well below FD truncation) does flag mismatches — confirms the validator is sensitive to its tolerance knob. Both build modes (default + `PULSIM_USE_AD_STAMP=ON`) pass: 301 cases / 4536 assertions each.
- Side note: `Circuit::devices_mutable()` accessor added to support the validation walk (stamps may mutate `pwl_state_`). Mutates only that member; structural device list is otherwise untouched.

## Phase 6: Performance
- [x] 6.1 Per-stamp micro-benchmark in [`test_ad_stamp_perf.cpp`](../../../core/tests/test_ad_stamp_perf.cpp): tight 100k-iteration loop on `MOSFET::stamp_jacobian` in saturation, with a 1k-iter warmup pass. The build flag `PULSIM_USE_AD_STAMP` selects which path is timed. Tagged `[.disabled-by-default]` so CI doesn't pay the cost on every run; explicit `[benchmark]` filter invokes it. Buck-circuit-level timing already lives in [`test_pwl_speedup_benchmark.cpp`](../../../core/tests/test_pwl_speedup_benchmark.cpp) for the PWL story; the per-stamp number isolates AD cost specifically.
- [ ] 6.2 AD inner-loop optimization — pending. The current `stamp_jacobian_via_ad` path heap-allocates a `std::vector<ADReal>` via `ad::seed_from_values` and the underlying `Eigen::AutoDiffScalar` allocates a fresh `Eigen::VectorXd` for each derivative. A pre-allocated thread-local arena, or specializing seed creation on a stack-sized derivative vector (`Eigen::Matrix<Real, N, 1>`), would eliminate both per-stamp heap touches. Logged as a follow-up.
- [x] 6.3 Per-stamp overhead measured on the development machine (AppleClang 17, Release+LTO, MOSFET in saturation):
  - Manual stamp: **222 ns / stamp**
  - AD stamp:     **848 ns / stamp**
  - Ratio:        **~3.81× overhead** (exceeds the 30% gate; ~280% above target).
  Total simulation-runtime overhead at the buck-converter level (already measured under [`test_pwl_speedup_benchmark.cpp`](../../../core/tests/test_pwl_speedup_benchmark.cpp)) shows the AD path completes within 3× the Behavioral wall-clock — much smaller relative overhead because stamping is only a fraction of the total step work. **Gate G.3 not yet met at the per-stamp level**; the 6.2 optimization is the path to closure. Recording the honest baseline so future PRs measure improvement against a known number.
- [x] 6.4 Structural bypass + assertion test in [`test_ad_stamp_perf.cpp`](../../../core/tests/test_ad_stamp_perf.cpp): each device's `stamp_jacobian_impl` dispatches to `stamp_jacobian_ideal` whenever `mode_ resolves to Ideal`, and the `#ifdef PULSIM_USE_AD_STAMP` only wraps the Behavioral branch. So PWL Ideal mode bypasses AD by construction — no runtime telemetry counter needed. The test pins this contract: a MOSFET in `SwitchingMode::Ideal` with committed `pwl_state_` produces J entries that match the Ideal stamp's pure-conductance form (`J(drain, drain) = +g_on`, `J(drain, source) = −g_on`, no gate coupling) regardless of build mode.
